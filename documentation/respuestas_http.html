El problema

Play Json Serialization

La libería base permite usar NamingConventions, pero no permite serializar non-sealed traits, ni permite unlimited case class parameters.

Debido a esto nos ha costado implementar NamingConventions en el proyecto.

En Play Json una NamingConvention permite definir el estilo de las claves de un Json. underscore, camelCase, etc.

Solución 1 – obsoleta

Usar Play Json NamingConventions.
Esto no permite serializar non-sealed traits, ni permite unlimited case class parameters.

Solución 2

No usar Play Json NamingConventions.

Aplicar una regex transformation al output de la serialización.

Por qué la solucion 2 es mejor

Porque modifica menos el proyecto, y permite al desarrollador liberarse de las restricciones impuestas por la librería estándar.
